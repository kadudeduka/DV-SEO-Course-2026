## Lab Steps

### Step 1: Select Content Pages to Evaluate

**What to Do:**
Select content pages to evaluate against Helpful Content principles:

1. **Select pages:**

### Step 2: Evaluate People-First Approach

**What to Do:**
Evaluate each content page for people-first approach:

1. **Assess people-first content:**
   - Is content created to help people, not just to rank?
   - Does content genuinely solve problems or answer questions?
   - Is content useful to users?
   - Does content serve users first, search engines second?

2. **Identify people-first signals:**
   - Content that solves real problems
   - Content that answers questions comprehensively
   - Content that provides genuine value
   - Content created for users, not algorithms

3. **Identify people-first gaps:**
   - Content created primarily to rank
   - Content that doesn't solve problems
   - Content that provides minimal value
   - Content optimized for search engines over users

4. **Document evaluation:**
   - For each page, assess people-first approach
   - Note signals and gaps
   - Rate people-first alignment

**Tool to Use:**
Browser (Review Content), Documentation Tool

**What to Record:**
Record people-first evaluation in the Submission Template, Section 2, Evidence Item 1:
- For each page: people-first assessment
- People-first signals identified
- People-first gaps identified
- People-first alignment rating

**Checkpoint:**
You've completed this step when:
- [ ] You have evaluated people-first approach for all pages
- [ ] You have identified signals and gaps
- [ ] You have recorded evaluation in the Submission Template

### Step 6: Determine Improvement or Pruning Actions

**What to Do:**
Determine specific actions for each content page:

1. **Categorize content:**
   - **Improve:** Content that aligns with Helpful Content principles but needs enhancement
   - **Prune:** Content that violates Helpful Content principles and can't be improved
   - **Keep:** Content that strongly aligns with Helpful Content principles

2. **Define improvement actions:**
   - For content to improve, specify what needs enhancement
   - Define specific improvement steps
   - Note resources needed

3. **Define pruning rationale:**
   - For content to prune, explain why it should be removed
   - Note Helpful Content principle violations
   - Assess impact of pruning

4. **Document actions:**
   - Create action plan for each page
   - Prioritize improvements
   - Document pruning decisions

**Tool to Use:**
Documentation Tool, Evaluation from previous steps

**What to Record:**
Record actions in the Submission Template, Section 3:
- For each page: improvement/pruning/keep decision
- Specific improvement actions (if applicable)
- Pruning rationale (if applicable)
- Prioritized action plan

## Troubleshooting

### Issue: I don't have enough content on my WordPress blog

**Symptoms:**
Your WordPress blog doesn't have enough pages to evaluate.

**Solution:**
1. Evaluate available pages even if there are only a few
2. Use provided content examples for additional practice
3. Focus on understanding the Helpful Content evaluation framework
4. Document that you used provided examples if needed

**If This Doesn't Work:**
Contact your instructor for provided content examples you can use for Helpful Content evaluation practice.

---

### Issue: I'm not sure if content is people-first

**Symptoms:**
You're uncertain about how to assess people-first approach.

**Solution:**
1. Review Day 13 Chapter 1 for Helpful Content principles
2. Ask: Was this content created to help people or to rank?
3. Does the content genuinely solve problems or answer questions?
4. Is the content useful to users?
5. When in doubt, document your reasoning

**If This Doesn't Work:**
Focus on understanding the Helpful Content evaluation framework conceptually. The learning is in the evaluation process, not perfect content assessment.

---

### Issue: I can't determine if content should be improved or pruned

**Symptoms:**
You're uncertain about whether to improve or prune content.

**Solution:**
1. Review Day 13 Chapter 1 for improvement vs pruning guidance
2. **Improve:** Content that aligns with principles but needs enhancement
3. **Prune:** Content that violates principles and can't be improved
4. When in doubt, document your reasoning
5. Focus on understanding the decision framework

**If This Doesn't Work:**
Focus on understanding the improvement/pruning decision framework conceptually. The learning is in the evaluation process, not perfect decision-making.
