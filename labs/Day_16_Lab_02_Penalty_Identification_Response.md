### Step 3: Identify Penalty Type (Algorithmic vs Manual)

**What to Do:**
Identify whether issues are algorithmic or manual:

1. **Check for manual action indicators:**
   - **Manual action:** Explicit notification in Search Console
   - **Manual action:** Specific violation mentioned
   - **Manual action:** Requires reconsideration request
   - **Manual action:** Human reviewer identified issue

2. **Check for algorithmic hit indicators:**
   - **Algorithmic hit:** No explicit notification
   - **Algorithmic hit:** Traffic/ranking drops without manual action message
   - **Algorithmic hit:** Correlates with algorithm update
   - **Algorithmic hit:** Requires quality improvements, not reconsideration

3. **Distinguish penalty types:**
   - If manual action message exists → Manual action
   - If no message but traffic/ranking drops → Likely algorithmic hit
   - If correlates with update → Likely algorithmic hit
   - If requires reconsideration → Manual action

4. **Document penalty type identification:**
   - Record penalty type identified (algorithmic or manual)
   - Note identification reasoning
   - Document indicators found
   - Create penalty type summary

## Troubleshooting

2. Focus on understanding the identification framework

---

### Issue: I can't distinguish manual actions from algorithmic hits

**Symptoms:**
You're uncertain about penalty type identification.

**Solution:**
1. Review Day 16 Chapter 1 for penalty type guidance
2. **Manual action:** Explicit notification in Search Console, requires reconsideration
3. **Algorithmic hit:** No explicit notification, requires quality improvements
4. Check for explicit messages (manual) vs no messages (algorithmic)
5. When in doubt, document your reasoning

**If This Doesn't Work:**
Focus on understanding the penalty type framework conceptually. The learning is in the identification process, not perfect penalty type identification.

---

### Issue: I'm not sure how to plan appropriate responses

**Symptoms:**
You're uncertain about response planning.

**Solution:**
1. Review Day 16 Chapter 1 for response guidance
2. **Manual actions:** Fix violations, submit reconsideration request
3. **Algorithmic hits:** Improve quality, wait for algorithm re-evaluation
4. Set realistic timelines (weeks to months for both)
5. When in doubt, document your reasoning

**If This Doesn't Work:**
Focus on understanding the response framework conceptually. The learning is in the response planning process, not perfect response plans.
