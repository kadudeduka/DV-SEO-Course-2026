## Tool Categories

Tools are classified into the following categories:

- **Search/Discovery:** Tools for finding, searching, or discovering information
- **Documentation/Reference:** Official documentation and reference resources
- **Analysis/Computation:** Tools for analyzing data or performing computations
- **Visualization:** Tools for creating or viewing visual representations
- **Data Collection:** Tools for gathering or collecting data
- **Content Creation:** Tools for creating content
- **Collaboration:** Tools for collaborative work
- **Automation:** Tools for automating tasks
- **AI-Assisted Reasoning:** Tools that use AI to assist with reasoning
- **Code Execution:** Tools for running or executing code
- **Monitoring/Inspection:** Tools for monitoring or inspecting systems
- **Storage/Management:** Tools for storing or managing data

## Registered Tools

### Google Search

**Tool Name:** Google Search  
**Category:** Search/Discovery  
**Access Type:** Free  
**Access Requirements:** None (accessible via any web browser)  
**URL:** https://www.google.com  
**Alternative Tools:** Bing Search, DuckDuckGo

**Description:**  
Google Search is the primary tool for observing real SERPs and understanding the modern search landscape. It enables learners to see how search engines present results to users and observe organic results, paid ads, and SERP features in their natural context.

**Professional Relevance:**  
Google Search is the primary tool for SERP analysis in professional SEO work. Entry-level SEO Analysts and SEO Executives regularly use Google Search to analyze search results.

**Used In:**
- Day 1, Lab 1 — Exploring the Real Search Landscape

## Tool Registration Process

### When to Register a Tool

Register a tool when:
- It is assigned in a validated lab requirements document
- It is used in a production-ready lab
- It is identified as a professional tool relevant to the domain

### How to Register a Tool

1. **Extract tool information** from validated lab requirements
2. **Classify tool** into appropriate category
3. **Document access requirements** and limitations
4. **Add tool entry** to this registry
5. **Link tool** to labs where it's used
6. **Note alternatives** if applicable

### Tool Entry Format

```markdown
### [Tool Name]

**Tool Name:** [Full tool name]  
**Category:** [Category from list above]  
**Access Type:** [Free/Paid/Limited Access/Provided Access]  
**Access Requirements:** [Specific requirements or "None"]  
**URL:** [Tool URL]  
**Alternative Tools:** [List alternatives if applicable]

**Description:**  
[2-3 sentences describing what the tool is and why it's used]

**Professional Relevance:**  
[1-2 sentences explaining how this tool is used in professional practice]

**Used In:**
- [Day X, Lab Y — Lab Title]
- [Additional labs if applicable]

**Registration Date:** [Date]  
**Registered By:** [Name]
```
