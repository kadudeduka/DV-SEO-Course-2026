## Lab Steps

### Step 1: Compile Performance Issues from Audit

**What to Do:**
Compile all performance issues identified in Lab 1 (or from provided examples):

1. **Review Lab 1 results:**
   - List all performance issues found
   - Include issues affecting LCP, INP, and CLS
   - Note which pages each issue affects
   - Document issue descriptions

2. **Organize issues:**
   - Group issues by Core Web Vitals (LCP, INP, CLS)
   - Group issues by type (server, images, JavaScript, CSS, etc.)
   - Create comprehensive issue list

3. **Document issues:**
   - Create list of all performance issues
   - Include issue description, affected pages, and Core Web Vitals impact

**Tool to Use:**
Lab 1 Submission Template or Provided Examples, Documentation Tool

**What to Record:**
Record issue compilation in the Submission Template, Section 1:
- Complete list of performance issues from audit
- Issue categorization (LCP, INP, CLS related)
- Pages affected by each issue
- Initial observations about issues

**Checkpoint:**
You've completed this step when:
- [ ] You have compiled all performance issues
- [ ] You have organized issues by type
- [ ] You have recorded issues in the Submission Template

---

### Step 2: Assess SEO Impact for Each Issue

**What to Do:**
For each performance issue, assess SEO impact using the framework from Day 10 Chapter 2:

1. **Assess Core Web Vitals impact:**
   - Does this issue affect LCP? (High/Medium/Low impact)
   - Does this issue affect INP? (High/Medium/Low impact)
   - Does this issue affect CLS? (High/Medium/Low impact)
   - Overall Core Web Vitals impact

2. **Assess ranking signal impact:**
   - How does this issue affect page experience signals?
   - How does this issue affect user behavior signals?
   - Overall SEO impact assessment

3. **Categorize by SEO impact:**
   - High SEO impact: Issues that significantly affect Core Web Vitals
   - Medium SEO impact: Issues that moderately affect Core Web Vitals
   - Low SEO impact: Issues with minor Core Web Vitals impact

**Tool to Use:**
Documentation Tool, Day 10 Chapter 2 knowledge

**What to Record:**
Record SEO impact assessment in the Submission Template, Section 2, Evidence Item 1:
- For each issue: SEO impact assessment (High/Medium/Low)
- Core Web Vitals affected
- Ranking signal impact
- Justification for impact rating

**Checkpoint:**
You've completed this step when:
- [ ] You have assessed SEO impact for all issues
- [ ] You have categorized issues by SEO impact
- [ ] You have recorded assessments in the Submission Template

---

### Step 3: Assess User Experience Impact for Each Issue

**What to Do:**
For each performance issue, assess user experience impact:

1. **Assess UX impact:**
   - How does this issue affect perceived loading speed?
   - How does this issue affect interactivity?
   - How does this issue affect visual stability?
   - Overall user experience impact

2. **Assess behavior signal impact:**
   - How does this issue affect bounce rates?
   - How does this issue affect time on page?
   - How does this issue affect user engagement?
   - Overall behavior signal impact

3. **Categorize by UX impact:**
   - High UX impact: Issues that significantly affect user experience
   - Medium UX impact: Issues that moderately affect user experience
   - Low UX impact: Issues with minor user experience impact

**Tool to Use:**
Documentation Tool, Day 10 Chapter 2 knowledge

**What to Record:**
Record UX impact assessment in the Submission Template, Section 2, Evidence Item 2:
- For each issue: UX impact assessment (High/Medium/Low)
- User experience factors affected
- Behavior signal impact
- Justification for impact rating

**Checkpoint:**
You've completed this step when:
- [ ] You have assessed UX impact for all issues
- [ ] You have categorized issues by UX impact
- [ ] You have recorded assessments in the Submission Template

---

### Step 4: Estimate Implementation Effort

**What to Do:**
For each performance issue, estimate implementation effort:

1. **Assess effort required:**
   - How much time is needed to fix this issue?
   - What resources are needed (developer, tools, etc.)?
   - How complex is the fix?
   - Overall effort assessment

2. **Categorize by effort:**
   - Low effort: Quick fixes (image optimization, CSS minification)
   - Medium effort: Moderate fixes (server optimization, code changes)
   - High effort: Complex fixes (architecture changes, major refactoring)

3. **Consider dependencies:**
   - Are there dependencies that affect effort?
   - Does this fix require other fixes first?
   - Are there resource constraints?

**Tool to Use:**
Documentation Tool, Day 10 Chapter 2 knowledge

**What to Record:**
Record effort estimation in the Submission Template, Section 2, Evidence Item 3:
- For each issue: effort assessment (High/Medium/Low)
- Estimated time required
- Resources needed
- Complexity assessment
- Justification for effort rating

**Checkpoint:**
You've completed this step when:
- [ ] You have estimated effort for all issues
- [ ] You have categorized issues by effort
- [ ] You have recorded estimations in the Submission Template

---

### Step 5: Create Prioritization Matrix

**What to Do:**
Create a prioritization matrix using impact vs effort:

1. **Plot issues on matrix:**
   - X-axis: Implementation Effort (Low to High)
   - Y-axis: Combined Impact (SEO + UX, Low to High)
   - Place each issue on the matrix

2. **Identify priority categories:**
   - **Priority 1 (Quick Wins):** High Impact, Low Effort
   - **Priority 2 (High Value):** High Impact, Medium/High Effort
   - **Priority 3 (Easy Improvements):** Medium Impact, Low Effort
   - **Priority 4 (Nice-to-Have):** Medium/Low Impact, Medium/High Effort

3. **Rank issues within priorities:**
   - Within each priority category, rank by impact
   - Consider dependencies between fixes
   - Create final prioritized list

**Tool to Use:**
Documentation Tool

**What to Record:**
Record prioritization matrix in the Submission Template, Section 3:
- Prioritization matrix (visual or table format)
- Issue placement on matrix
- Priority category for each issue
- Final prioritized ranking

**Checkpoint:**
You've completed this step when:
- [ ] You have created prioritization matrix
- [ ] You have categorized issues by priority
- [ ] You have ranked issues within priorities
- [ ] You have recorded matrix in the Submission Template

---

### Step 6: Create Prioritized Action Plan

**What to Do:**
Create a detailed, prioritized action plan:

1. **List prioritized fixes:**
   - Start with Priority 1 (Quick Wins)
   - Then Priority 2 (High Value)
   - Then Priority 3 (Easy Improvements)
   - Finally Priority 4 (Nice-to-Have)

2. **For each fix, specify:**
   - What needs to be fixed
   - How to fix it (specific steps)
   - Who needs to be involved (if applicable)
   - Estimated time and resources
   - Expected impact

3. **Create implementation timeline:**
   - When each fix should be implemented
   - Dependencies between fixes
   - Resource allocation

**Tool to Use:**
Documentation Tool

**What to Record:**
Record action plan in the Submission Template, Section 3:
- Prioritized fix list (all issues ranked)
- Specific actions for each fix
- Implementation timeline
- Resource requirements
- Expected impact summary

**Checkpoint:**
You've completed this step when:
- [ ] You have created prioritized action plan
- [ ] Fixes are ranked and detailed
- [ ] Timeline is realistic
- [ ] You have recorded plan in the Submission Template

### Issue: I'm not sure how to assess impact

**Symptoms:**
You're uncertain about how to rate SEO or UX impact.

**Solution:**
1. Review Day 10 Chapter 2 for impact assessment guidance
2. High impact: Issues that significantly affect Core Web Vitals or user experience
3. Medium impact: Issues that moderately affect Core Web Vitals or user experience
4. Low impact: Issues with minor effects
5. When in doubt, document your reasoning

**If This Doesn't Work:**
Focus on understanding the prioritization framework conceptually. The learning is in the prioritization process, not perfect impact assessment.
