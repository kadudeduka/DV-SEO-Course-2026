## Lab Steps

### Step 1: Select Content to Optimize

**What to Do:**
Select an existing blog post from your WordPress blog to optimize. Choose a post that:
- Is at least 500 words long
- Has room for improvement (structure, headings, formatting)
- Covers a topic you understand well
- Has not been heavily optimized before

If you don't have a suitable post, create a new 500+ word post on a topic of your choice, then optimize it.

**Tool to Use:**
WordPress Blog Editor

**What to Record:**
Record your selection in the Submission Template, Section 1:
- Post title and URL
- Original word count
- Topic/niche

**Checkpoint:**
You've completed this step when:
- [ ] You have selected or created a blog post
- [ ] The post is at least 500 words
- [ ] You have recorded your selection in the Submission Template

### Step 2: Analyze Current Content Structure

**What to Do:**
Analyze your selected post's current structure using browser developer tools:
1. Open your blog post in a browser
2. Right-click and select "Inspect" (or press F12)
3. Examine the HTML structure:
   - Check heading hierarchy (H1, H2, H3)
   - Identify paragraph structure
   - Note any lists or formatting
   - Check for internal links

Document what you find:
- Current heading structure (are headings logical and hierarchical?)
- Paragraph length (are paragraphs too long or too short?)
- Formatting usage (lists, bold text, etc.)
- Internal linking (are there internal links? Are they relevant?)

**Tool to Use:**
Browser Developer Tools (Inspect Element)

**What to Record:**
Record your analysis in the Submission Template, Section 2, Evidence Item 1:
- Screenshot of browser inspection showing HTML structure
- Document current heading hierarchy
- Note paragraph structure observations
- List formatting elements found
- Note internal linking status

**Checkpoint:**
You've completed this step when:
- [ ] You have inspected the HTML structure
- [ ] You have documented current heading hierarchy
- [ ] You have noted structure observations
- [ ] You have recorded findings in the Submission Template

### Step 3: Optimize Content Structure (Headings)

**What to Do:**
Improve your post's heading structure:
1. Ensure there is one H1 heading (usually the post title)
2. Organize content with logical H2 headings for main sections
3. Use H3 headings for subsections under H2s
4. Ensure headings are descriptive and reflect content
5. Maintain logical hierarchy (H1 → H2 → H3)

Make these changes in your WordPress editor:
- Add or improve H2 headings for main sections
- Add H3 headings for subsections where needed
- Ensure headings accurately describe the content below them
- Remove any heading hierarchy issues (e.g., H3 without H2)

**Tool to Use:**
WordPress Blog Editor

**What to Record:**
Record your changes in the Submission Template, Section 2, Evidence Item 2:
- Screenshot of optimized heading structure in WordPress editor
- List of headings added or modified
- Explanation of heading hierarchy improvements
- Before/after comparison of heading structure

**Checkpoint:**
You've completed this step when:
- [ ] You have improved heading hierarchy
- [ ] You have logical H1 → H2 → H3 structure
- [ ] Headings accurately describe content
- [ ] You have documented changes in the Submission Template

---

### Step 4: Optimize Keyword Placement

**What to Do:**
Improve natural keyword placement in your content:
1. Identify the primary keyword or topic of your post
2. Ensure the keyword appears naturally in:
   - The H1 heading (post title)
   - At least one H2 heading
   - Naturally in the first paragraph
   - Naturally throughout the content where it makes sense
3. Avoid keyword stuffing:
   - Don't force keywords into every sentence
   - Don't use awkward phrasing to include keywords
   - Keywords should support understanding, not dominate content

Make these changes in your WordPress editor:
- Review keyword placement throughout the post
- Add keywords naturally where they support understanding
- Remove any forced or awkward keyword usage
- Ensure keywords appear in key locations (title, headings, first paragraph)

**Tool to Use:**
WordPress Blog Editor

**What to Record:**
Record your optimization in the Submission Template, Section 2, Evidence Item 3:
- Screenshot showing keyword placement in key locations
- List of keyword placement improvements made
- Explanation of how keywords support understanding
- Note on avoiding keyword stuffing

**Checkpoint:**
You've completed this step when:
- [ ] Keywords appear naturally in key locations
- [ ] No keyword stuffing is present
- [ ] Keywords support understanding
- [ ] You have documented keyword placement in the Submission Template

4. Ensure links are contextually relevant (they add value to readers)

Make these changes in your WordPress editor:
- Find related posts or pages on your blog

- Place links where they naturally fit in the content
- Ensure links provide value to readers

## Troubleshooting

### Issue: I don't have a WordPress blog post to optimize

**Symptoms:**
You don't have an existing blog post that meets the requirements.

**Solution:**
1. Create a new 500+ word blog post on a topic you know well
2. Write the post first (don't worry about optimization initially)
3. Then proceed with the optimization steps
4. This approach actually demonstrates optimization principles well—you'll see the difference optimization makes

**If This Doesn't Work:**
Contact your instructor for a provided example post you can optimize.

---

### Issue: Browser developer tools are confusing

**Symptoms:**
You're not sure how to use browser inspection tools or find them overwhelming.

**Solution:**
1. You only need basic inspection—right-click on your content and select "Inspect"
2. Look for heading tags (H1, H2, H3) in the HTML
3. You don't need to understand all HTML—just identify headings and structure
4. Focus on what you can see in the visual content, then check the HTML structure

**If This Doesn't Work:**
You can complete the lab by working directly in WordPress editor and documenting changes visually. Browser inspection is helpful but not strictly required if you can document structure changes in WordPress.

---

### Issue: I'm not sure if my keyword placement is natural

**Symptoms:**
You're unsure whether keywords appear naturally or if you're forcing them.

**Solution:**
1. Read your content aloud—does it sound natural?
2. Ask: Would I use this phrasing if I wasn't optimizing for SEO?
3. If keywords feel forced, remove them or rephrase
4. Remember: Keywords should support understanding, not dominate content

**If This Doesn't Work:**
Focus on ensuring content is genuinely useful first, then add keywords where they naturally fit. Quality content with natural keyword usage is better than optimized content with forced keywords.

---

### Issue: I can't find related content for internal links

**Symptoms:**
You don't have other blog posts or pages to link to.

**Solution:**
1. Link to your blog's homepage or category pages
2. Link to your "About" page or other static pages
3. If you have other posts (even on different topics), link to them if contextually relevant
4. The goal is to practice internal linking—even a few links demonstrate the concept

**If This Doesn't Work:**
Document that you understand internal linking principles and would add links when you have related content. The learning objective is understanding, not having perfect internal linking.
